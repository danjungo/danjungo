clean_column_value(v)

There are two basic ways to clean a column‘s value: h() and sanitize(). The latter is useful when the column contains valid html data, and you want to just disable any scripting. People can always use field overrides to clean data one way or the other, but having this override lets people decide which way it should happen by default.

Why is it not a configuration option? Because it seems like a somewhat rare request. But it could eventually be an option in config.list (and config.show, I guess).

 6:         # check for an override helper
 7:         value = if column_override? column
 8:           # we only pass the record as the argument. we previously also passed the formatted_value,

10:           # performance of our default formatting. see issue #138.
11:           send(column_override(column), record)
12:         elsif column.inplace_edit
13:           active_scaffold_inplace_edit(record, column)
14:         else
15:           value = record.send(column.name)
16: 
17:           if column.association.nil? or column_empty?(value)
18:             formatted_value = clean_column_value(format_column(value))
19:           else
20:             case column.association.macro
21:               when :has_one, :belongs_to
22:                 formatted_value = clean_column_value(format_column(value.to_label))
23: 
24:               when :has_many, :has_and_belongs_to_many
25:                 firsts = value.first(4).collect { |v| v.to_label }
26:                 firsts[3] = '…' if firsts.length == 4
27:                 formatted_value = clean_column_value(format_column(firsts.join(', ')))
28:             end
29:           end
30: 
31:           formatted_value
32:         end
33:

36:       end
