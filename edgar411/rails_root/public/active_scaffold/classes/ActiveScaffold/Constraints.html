apply_constraints_to_record(record)

Applies constraints to the given record.

Searches through the known columns for association columns. If the given constraint is an association, it assumes that the constraint value is an id. It then does a association.klass.find with the value and adds the associated object to the record.

conditions_from_constraints()

Returns search conditions based on the current scaffold constraints.

Supports constraints based on either a column name (in which case it checks for an association or just uses the search_sql) or a database field name.

All of this work is primarily to support nested scaffolds in a manner generally useful for other embedded scaffolds.

     # File vendor/plugins/active_scaffold/lib/constraints.rb, line 40
 40:     def conditions_from_constraints
 41:       conditions = nil
 42:       active_scaffold_constraints.each do |k, v|
 43:         column = active_scaffold_config.columns[k]
 44:         constraint_condition = if column
 45:           # Assume this is a multi-level association constraint.
 46:           # example:
 47:           #   data model: Park -> Den -> Bear
 48:           #   constraint: :den => {:park => 5}
 49:           if v.is_a? Hash
 50:             far_association = column.association.klass.reflect_on_association(v.keys.first)
 51:             field = far_association.klass.primary_key
 52:             table = far_association.table_name
 53: 
 54:             active_scaffold_joins.concat([{k => v.keys.first}]) # e.g. {:den => :park}
 55:             constraint_condition_for("#{table}.#{field}", v.values.first)
 56: 
 57:           # If a column is an association, then we do NOT want to use .search_sql. If anything,
 58:           # search_sql will refer to a human-searchable value on the associated record.
 59:           elsif column.association
 60:             # when the reverse association is a :belongs_to, the id for the associated object only exists as
 61:             # the primary_key on the other table. so for :has_one and :has_many (when the reverse is :belongs_to),
 62:             # we have to use the other model's primary_key.
 63:             #
 64:             # please see the relevant tests for concrete examples.
 65:             field = if [:has_one, :has_many].include?(column.association.macro)
 66:               column.association.klass.primary_key
 67:             else
 68:               column.association.options[:association_foreign_key] || column.association.options[:foreign_key] || column.association.association_foreign_key
 69:             end
 70: 
 71:             table = case column.association.macro
 72:               when :has_and_belongs_to_many
 73:               column.association.options[:join_table]
 74: 
 75:               when :belongs_to
 76:               active_scaffold_config.model.table_name
 77: 
 78:               when :has_many
 79:               column.association.table_name
 80: 
 81:               else
 82:               column.association.table_name
 83:             end
 84: 
 85:             active_scaffold_joins.concat column.includes
 86:             constraint_condition_for("#{table}.#{field}", v)
 87:           elsif column.searchable?
 88:             active_scaffold_joins.concat column.includes
 89:             constraint_condition_for(column.search_sql, v)
 90:           end
 91:         elsif active_scaffold_config.model.column_names.include? k.to_s
 92:           constraint_condition_for(k.to_s, v)
 93:         else
 94:           raise ActiveScaffold::MalformedConstraint, constraint_error(k), caller
 95:         end
 96: 
 97:         conditions = merge_conditions(conditions, constraint_condition)
 98:       end
 99: 
100:       conditions
101:     end

register_constraints_with_action_columns()

For each enabled action, adds the constrained columns to the ActionColumns object (if it exists). This lets the ActionColumns object skip constrained columns.

If the constraint value is a Hash, then we assume the constraint is a multi-level association constraint (the reverse of a has_many :through) and we do NOT register the constraint column.
